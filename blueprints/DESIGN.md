# DESIGN — Global taste + system

Applies to every page generated by WRKS Studio. Overrides in per-page
blueprints only when explicitly noted. Extracted + refined from the
`~/.claude/global-memory/feedback_*` taste-profile files.

## Philosophy

Editorial magazine restraint. Aesop / Mercury / Linear / Stripe Press
territory. Not Notion-playful, not Wix-generic, not consulting-firm blue.
The site should feel like an art director laid it out, not an AI templated
it. Real over decorative. Restraint over ornament.

## Non-negotiable BANS

Reject any output containing these:

**Typography bans**
- Italics of any kind (no `italic` class, no `font-style: italic`, no
  italic-variant typefaces, no `<em>` / `<i>` for emphasis)
- Uppercase inside body copy (mono-caps only for eyebrow labels)
- Comic Sans, Papyrus, Impact, any script/handwriting except signature use
- Default `font-family: sans-serif` — always name a real typeface

**Color bans**
- Pure `#000000` and `#ffffff` at large fills (use `#0a0a0c` / `#f5f0e6`
  or the palette's neutrals)
- AI-signature purple → blue → teal gradients (rgba(139,92,246) → rgba(59,130,246))
- Neon oversaturated colors (`>85%` saturation on primary)
- More than one accent color visible per section
- Rainbow multi-color palettes; pick a hue family

**Layout bans**
- Centered heroes as the default (asymmetric editorial only)
- Three-column identical card grids without contrast in size or emphasis
- `h-screen` / `100vh` / `100dvh` — use fixed px `min-height`
- Boxed cards for EVERYTHING (pick form factor: sphere, arc, ribbon,
  hairline-divided rows — mix in cards where they earn it)
- Full-viewport hero images (bleed the right column, keep copy on left)

**Motion bans**
- `scale()` on hover greater than 1.02
- Bounce cubic-beziers on primary CTAs (linear-app style, ease-out only)
- Parallax on above-fold content
- Auto-play video with sound
- Infinite spinning icons / orbits

**Copy bans (write these NOWHERE)**
- Elevate, Unlock, Unleash, Empower, Seamless, Seamlessly,
  Revolutionary, Cutting-edge, Leverage, Next-gen, Game-changer,
  Ecosystem (as filler), Journey (as filler), Transform (as filler)
- "In today's fast-paced world"
- "We understand that…"
- Fabricated stats without attribution (no "increases conversions by 47%"
  unless the user's data literally says so)
- Emoji in headlines / CTAs / body copy
- Multi-claim headlines ("The best X for Y that also does Z") — one claim

**Meta-cruft bans** (never render as decoration)
- "EST. 2026" pills
- Roman numerals as section labels ("N° 01 — INTRODUCTION")
- Decorative-only mono caps ("THE BRIEF", "OUR PROCESS" when there's no
  section they lead into)
- Small circle/orb decorations before wordmarks
- Section eyebrows on every headline (use only when nav context is thin)

## Required patterns

Ship every output with these:

**Typography stack (single family discipline)**
- Display: Fraunces (variable), Cabinet Grotesk, Instrument Serif, or
  Playfair (pick ONE per site based on voice_descriptor)
- Body: Geist Sans, Inter, Söhne, or Space Grotesk (pick ONE per site)
- Mono: Geist Mono, JetBrains Mono, IBM Plex Mono (for labels + code)
- Load via `<link rel="preload">` from Google Fonts
- Never use both a serif display AND a serif body

**Type scale (rem-based, must be present)**
- Display / h1: 60-80px (clamp 60-80), weight 500-600, tracking -0.03em, leading 1.04
- Section heading / h2: 36-48px, weight 500-600, tracking -0.02em, leading 1.1
- Card title / h3: 20-24px, weight 600, tracking -0.015em, leading 1.2
- Body: 16px (min), weight 400, tracking -0.003em, leading 1.6
- Lead / subhead: 18-19px, weight 400, leading 1.55
- Small / caption: 13.5px, weight 500
- Mono eyebrow: 11.5px, weight 500, tracking 0.14em, uppercase

**Color palette (5 tokens, all named)**
```
primary       #hex   — most visible accent, hero CTAs, key metrics
secondary     #hex   — dark ink for body text, section borders
tertiary      #hex   — quiet accent, quote treatment, decorative
neutral-warm  #hex   — cream/bone bg for light sections (never pure white)
neutral-dark  #hex   — near-black for dark sections (never pure black)
```
With scale ramps: 5 stops each `100/300/500/700/900` (light → dark).
Rules:
- ≤1 accent color visible per section
- Saturation ceiling 78% for primary, 32% for tertiary
- Palette source: HARD-CONSTRAINED to extracted brand palette when ingest
  provides one. Otherwise Opus picks per voice_descriptor.

**Button styling**
- Primary CTA on light: solid neutral-dark bg + neutral-warm ink, pill or
  10px radius, 14px vertical / 26px horizontal padding, 15px semibold
- Primary CTA on dark: solid `#ffffff` bg + neutral-dark ink (same
  dimensions)
- Secondary CTA: transparent + 1px border matching text color, no fill
- Hover: 1px `translateY(-1)`, no scale, brighten fill 4-8%, 200ms ease-out
- Active: `translateY(0)`, inset shadow, 60ms snap
- Radius options: 6px (sharp), 12px (default), 999px (pill) — pick one per
  site, use consistently

**Spacing scale (px, USE ONLY THESE VALUES)**
`4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48 / 64 / 88 / 100 / 120 / 168`
No other values. Ever.

**Layout tokens**
- Container max-width: 1180px (or 1440px for dense info sites)
- Container horizontal padding: 40px (24px on mobile <768px)
- Section vertical rhythm:
  - Light sections: 100px top / 100px bottom
  - Dark sections: 120px both (larger to breathe)
  - Hero: 168px top (clear floating nav) / 128px bottom, min-height 720px
- Grid base: 12 columns, 24px gutter

**Motion tokens**
- Enter: spring physics, stiffness 100, damping 20
- Hover: 200ms ease-out `[0.2, 0, 0, 1]`
- Layout shifts: 300ms cubic-bezier `[0.4, 0, 0.2, 1]`
- Only `transform` and `opacity` — never animate width/height/margin

**Shadow recipes**
- Card resting: `0 1px 3px rgba(0,0,0,0.04), 0 20px 44px -20px rgba(0,0,0,0.12)`
- Card hover: same but `-16` blur (deeper)
- Elevated (modal, floating nav): `0 12px 34px rgba(0,0,0,0.35)` + inset
  `0 1px 0 rgba(255,255,255,0.9)` when on white
- Never pure `#000` shadows — always cool-tinted `rgba(20,30,60, x)` or
  neutral-dark-tinted

**Icon system**
- Stroke-only (1.7px stroke width, Feather / Lucide style) unless
  voice_descriptor is `playful` (then duotone)
- 16px / 20px / 24px sizes, no fractional
- Never emoji in place of icons

## Editorial composition rules

- **Asymmetric heroes**: copy left rail (60% width, max 780px) +
  portrait/hero-image right column (40%, full-bleed to edge). Never
  center-stack.
- **Section rhythm**: alternate light and dark sections. Long runs of
  same-tone sections lose the eye.
- **Whitespace as design**: 88-120px between sections. Cramped is
  cheaper-looking than sparse.
- **One focal element per section**: don't compete headlines with images.
  Either the headline is huge and image is quiet, or the image is hero
  and the headline is small caps.
- **Progressive disclosure**: state → context → action, not action → context → state.
- **Photography over illustration**: real photos beat illustrated icons
  every time. When illustrations are used, keep them monochromatic and
  strokey.

## Accessibility (baked in, non-optional)

- Every image: `alt=""` (empty for decorative) or descriptive alt
- Every button: text label (no icon-only buttons without `aria-label`)
- Color contrast: WCAG AA minimum (4.5:1 body, 3:1 large text)
- Focus states: 2px outline in `primary` with 2px offset
- Motion: respect `prefers-reduced-motion` (disable spring physics)
- Skip-to-content link at top of `<body>`
- Semantic HTML: `<header>`, `<main>`, `<section>`, `<nav>`, `<footer>`

## HTML output rules (for LLM generation)

- Emit ONE complete HTML document per page (doctype → html → head → body)
- `<head>`: meta charset, viewport, title, description, og:image, og:title,
  favicon, preconnect Google Fonts + font stylesheet link
- Tailwind CDN in `<head>` (`https://cdn.tailwindcss.com`) for
  self-contained rendering
- Inline `<style>` in `<head>` for CSS variables (palette + type),
  overrides Tailwind for design tokens
- No external JS bundle (small `<script>` for menu toggle etc. inline)
- Semantic HTML5 tags
- All Tailwind classes; no external CSS file needed
- Every editable element gets `data-edit-id="section.slot"` for future
  inline editing
