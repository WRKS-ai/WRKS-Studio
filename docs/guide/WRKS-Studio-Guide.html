01 · The problem

There's a gap between
having an idea and having a real business.

Imagine your aunt has a great idea for a small bakery. To actually launch it, here's what she has to do:

  • Build a website (or hire someone to)
  • Write copy that actually sells
  • Set up Instagram, Facebook, maybe TikTok
  • Design ad creatives if she wants paying customers
  • Figure out SEO so people can find her
  • Set up Stripe to take payments
  • Connect a CRM so she can follow up with leads

Each one is a wall. Each wall takes time, money, or expertise she doesn't have. Most people don't make it past the third wall. The bakery idea dies — not because it was bad, but because the path to launching it was exhausting.

02 · The solution

WRKS Studio
replaces the walls with one agent.

Instead of forcing your aunt to learn seven tools, WRKS Studio gives her one personalised AI agent. She talks to it. It builds the website. It writes the posts. It designs the ad creative. It publishes everything. It remembers what works and gets sharper over time.

One person → one agent → many outputs
The key shift: WRKS Studio doesn't simplify the existing tools. It removes the need to use them at all. The user never opens Wix, never writes a caption, never logs into Google Ads. They talk. The agent does the rest.

04 · Meet the agent

It's not a chatbot.
It's a colleague.

The single biggest difference between WRKS Studio and every other AI tool is this: the agent has a personality. The user picks one of 3–6 personalities at signup. They give it a name. They choose its voice. Over time, it adapts to how they talk.

Why this matters for the business

After six months of using WRKS, the agent knows the user's voice, their offers, their style, what's worked, what's flopped. Switching to a competitor means losing a colleague who knows them. That's the retention design. People don't churn from a colleague — they churn from a tool.

Has a name

The user names it on signup. "Hey Nova, can you…"

Has a voice

Picked from ElevenLabs voices. Speaks the same on web and mobile.

Has a personality

Direct vs. encouraging, formal vs. casual, brief vs. detailed.

Has memory

Remembers every decision the user has ever approved or rejected.

What this isn't. Not a chatbot popup. Not "Hi, I'm a virtual assistant." The agent is named, voiced, and present on every screen. It's the centre of the product, not a feature added on the side.

05 · How it works (continued)

The hard rule:
show before you do.

Steps 3 and 5 are highlighted in the diagram for a reason. They are the "human gates" — the moments where the agent stops and waits for the user. This is not a quirk. It's a deliberate design choice and you will see it baked into the engineering.

Reversible vs. irreversible

Some actions can be undone (drafting copy, generating an image). Those can run automatically. Other actions cannot be undone (publishing to Instagram, deploying a live site, sending money via Stripe). Those require explicit user confirmation, every single time.

Reversible — can auto-run Irreversible — must confirm
Drafting a social post in the staging area Publishing it to Instagram
Generating new headline copy Deploying it to the live website
Designing an ad creative Forwarding form data to the user's CRM
Drafting a discount code Activating the discount code on Stripe
Why this matters to you. When you're building, the approval gates are not nice-to-haves. They are the trust mechanism that lets users hand the keys to an AI in the first place. Build them with care.

06 · What it builds

Five things.
Done exceptionally.

WRKS Studio doesn't try to do everything a business owner needs. It deliberately picks five high-impact deliverables and does them at a professional level. Anything outside this list is out of scope for version one.

01

Websites & Funnels

Landing pages, full sites, sales funnels. Forms forward to the user's CRM. Stripe payments embedded. Booking links embedded.

02

Social Content

Posts, captions, and visuals for Instagram, Facebook, and LinkedIn. Each platform's format rules built into the framework.

03

Ad Creatives

Headlines, body text, image direction. Designed to a proven ad framework. User downloads and uploads to Meta / Google Ads.

04

Copywriting

Page copy, CTAs, offer descriptions, brand voice writing. Not generic AI text — written to a conversion framework.

05

SEO & Blog

Long-form posts, metadata, keyword targeting, internal linking. Structured for search and for the user's brand.

What it deliberately doesn't do

No email automation. No sales sequences. No CRM build. Those are future versions. Saying no protects the focus.

07 · The three connections

Only three doors out.
No doors in.

WRKS Studio talks to the outside world in only three ways. And critically, all three are outbound only. The platform never reads live data from anywhere — no analytics, no follower counts, no revenue dashboards. This is deliberate.

All arrows point outward — nothing flows back in
Connection What it does What it never does
Social Publishes posts to Instagram, Facebook, LinkedIn Reads followers, engagement, or analytics
CRM Forwards form submissions to the user's CRM webhook Reads contacts, deals, or pipeline data
Stripe Embeds payment links into pages and funnels Reads transaction history or revenue
Why this is a feature, not a limitation. Reading live data means integrations, permissions, broken APIs, and complexity that slows down the wow moment. The agent doesn't need to read your analytics to write a good post. It needs to know your business — and it learns that by working with you, not by scraping a dashboard.

08 · The quality equation

The moat
is this single equation.

Anyone can spin up a ChatGPT wrapper and have it write Instagram captions. So why is WRKS Studio different? Because of this:

What's a "framework"?

A WRKS framework is the proprietary, step-by-step method WRKS has developed for making a specific deliverable type — built and validated on real client work. It's not a prompt. It's logic, structure, format rules, quality criteria, and execution sequence rolled together.

Think of it like a chef's recipe + their plating rules + their ingredient sourcing standards. A general AI can describe a dish. A WRKS framework knows how a real chef would make it.

What's "memory"?

Memory is everything WRKS knows about this specific user's business: their voice, their audience, their offers, their pricing, what's worked before, what they've rejected. It's built up over time, not by reading external data, but by being the agent that does the work.

Without framework: generic AI content. Sounds like every other startup.
Without memory: on-brand garbage. Sounds professional, but says nothing about the user's actual business.
Both together: output that's professionally correct AND personally relevant. That's the moat.

09 · The five layers

How the system is built,
top to bottom.

Engineering teams talk about systems in "layers." Top is what the user sees. Bottom is the plumbing. WRKS Studio has five.

The five-layer architecture

Read this like a building. The user lives on the top floor (Layer 1). Below them, the agent does the thinking (Layer 2). Below that, the staging environments are where deliverables come to life (Layer 3). Below that, the WRKS frameworks and execution engine produce the actual outputs (Layer 4). At the bottom, connections to the outside world ship things to where they need to go (Layer 5).

10 · Who it's for

Built for owners,
not for marketers.

WRKS Studio's user is a small business owner — a hairdresser, a bakery, a clothing brand, a personal trainer, a consultant. Someone who runs their own business and doesn't have time to learn marketing tools.

Where they actually use it

Not at a desk. Between clients. On the train. During lunch. WRKS Studio is mobile-first because that's where its users live. The web app is the full environment for deeper work, but most commands are fired from a phone in the real world.

The pricing model

No free tier. Free trial → paid. Pricing is gated only by how many active business profiles a user runs — never by features, rate limits, or AI credits.

11 · How it's different

Why this isn't
"just another AI tool."

Compared to What they do What WRKS does differently
ChatGPT General-purpose chat. Writes anything you ask. Knows your business. Outputs follow proven frameworks. Publishes the result.
Wix / Framer Website builders. You drag, drop, write, publish. You don't build. The agent builds and the user only directs and approves.
Canva Design tool. You make the visuals yourself. Visuals are produced as part of the deliverable, not a tool you operate.

The category we're inventing. WRKS Studio is not competing with any of these tools individually. It's replacing the need to use them. The pitch is: stop assembling a stack of tools. Talk to one agent.

12 · The tech stack, simplified

What we're using
to build it.

These are the tools we picked. You don't need to memorise these — just know what each one does and why it's there.

The core (everything sits on these)

Next.js
The framework that runs our website + product app. Pages, routing, server logic — all in one.
TypeScript
Adds types to JavaScript so we catch mistakes before they ship.
Tailwind
How we style things. Instead of writing CSS files, we use small utility classes inside the HTML.
shadcn/ui
Pre-made buttons, dialogs, forms, etc. We copy them into our project and own the code.
pnpm
The package manager. It installs all the libraries our code depends on. Faster than npm.
Turborepo
Lets us put multiple apps (marketing site, web app, mobile app) in one repository, with shared code.

The user system

Clerk
Handles signup, login, password reset. Also gives us "Organizations" — perfect for our business profiles.
Supabase
The database. Stores users, business profiles, memory, assets. Has built-in security so each business's data is isolated.

The intelligence layer

Claude API
The AI brain. We send it instructions, it returns text, plans, decisions. This is the agent's mind.
ElevenLabs
Gives the agent its voice. Converts text to speech and speech to text.

Resend

Helping Claude Code build (MCP servers)

shadcn MCP
Lets me browse and install shadcn components without leaving the chat.
21st.dev Magic

Context7
Gives me up-to-date docs for fast-moving frameworks.
GitHub MCP
I can manage pull requests and issues from inside our chat.
The shape to remember. Frontend = Next.js + Tailwind + shadcn. Backend = Supabase + Clerk. Brain = Claude + ElevenLabs. Hosting = Vercel + Cloudflare. That's it. Everything else slots into one of those buckets.
